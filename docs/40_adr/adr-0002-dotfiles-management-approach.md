# ADR-0002: ドットファイルの運用方式を再選定する

- Status: Accepted
- Date: 2025-12-21

## Context
- ドットファイルの反映方法としてシンボリックリンク以外の選択肢も把握した上で、方式を選び直したい。
- 目的は「履歴管理」「再現性」「移行の簡便さ」「運用負担の小ささ」。

## Decision
- **当面はシンボリックリンク方式を継続**する。
- ただし、将来的に規模が拡大した場合はツール導入（例: stow / chezmoi）を再検討する。

## Alternatives
1. **シンボリックリンク方式（現行）**
   - 長所: 仕組みが単純、依存が少ない、挙動が明確
   - 短所: 初回セットアップで退避作業が必要、手順の手動化が残る

2. **Dotfiles 管理ツール（stow / chezmoi など）**
   - 長所: 自動化が進み、環境差分の吸収が容易
   - 短所: 学習コストと依存が増える、ツール更新の追随が必要

3. **手動コピー運用**
   - 長所: 手順が直感的、ツール不要
   - 短所: 差分・履歴の追跡が煩雑、反映漏れが起きやすい

4. **スクリプトで自動配置**
   - 長所: 反映手順を自動化できる
   - 短所: スクリプト保守が必要、環境差異への対応が発生

5. **パッケージ管理に寄せる（例: Homebrew Bundle 等）**
   - 長所: 依存関係やツール導入まで含めた再現が可能
   - 短所: ドットファイル単体の管理とは目的がずれる

## Consequences
- シンプルさを優先した運用を継続できる。
- 反映手順の自動化は最低限に留める。
- 需要が増えた段階でツール方式へ移行できるよう、ADRで判断理由を残す。
